Abstract

A latência e a fragilidade das pipelines de dados tradicionais (ETL) representam o maior entrave para a gestão preditiva de crises hospitalares. Este documento detalha a infraestrutura de rede analítica que implementa o conceito de Zero Pipeline Integration. Ao acoplar motores de cálculo baseados em Detrended Fluctuation Analysis (DFA) diretamente aos repositórios de eventos de saúde, o sistema elimina etapas de processamento intermediário que distorcem a assinatura fractal dos dados. O estudo de caso em São Paulo demonstra que esta arquitetura de "fluxo puro" permite identificar a Lentidão Crítica hospitalar em tempo real, fornecendo uma janela de intervenção logística sem precedentes.

1. Infraestrutura Analítica e Zero Pipeline

A infraestrutura analítica proposta baseia-se na descentralização do processamento e na ingestão direta de micro-eventos hospitalares. Enquanto modelos tradicionais aguardam o fechamento de lotes (batches) mensais do SIH/SUS, o framework opera em uma topologia de Fluxo de Alta Fidelidade.

O Conceito de Zero Pipeline

Tradicionalmente, os dados passam por múltiplas transformações que inserem ruído artificial. O Zero Pipeline utiliza pesos probabilísticos para normalizar dados brutos em tempo real, permitindo que as métricas físicas (Entropia, Fisher, Lyapunov) reflitam a biologia do surto, não o erro do processo.

Gargalo Tradicional (ETL)

Pipelines estáticos que causam latência e perda de transientes rápidos. Impossibilitam a deteção de 'flickering' epidemiológico.

Arquitetura Zero Pipeline

Conectividade síncrona com fontes SIH. Análise imediata de memória fractal e reconstrução de atratores em tempo real.

2. Fundamentação Matemática e Física

Epidemias são fenômenos de auto-organização crítica. A transição de um estado estável para um surto exponencial é, matematicamente, uma bifurcação catastrófica.

2.1. Termodinâmica de Estados Meta-estáveis

Utilizamos a distribuição de Boltzmann para inferir o potencial efetivo do sistema hospitalar. Se \( P(r) \) é a densidade de probabilidade das variações de internação, o potencial \( U(r) \) é:

$$U(r) = - \frac{\sigma^2}{2} \ln(P(r) + \epsilon)$$

Um poço de potencial "raso" indica que o sistema hospitalar perdeu sua barreira de resiliência e pode colapsar sob qualquer choque estocástico mínimo.

2.2. Lentidão Crítica e Sinais Antecedentes

À medida que o sistema de saúde aproxima-se de um Tipping Point, o tempo de recuperação após uma flutuação aumenta. Este fenômeno de Critical Slowing Down é capturado pela Autocorrelação Lag-1 (\( AC(1) \)):

$$AC(1) = \frac{\sum_{t=2}^N (r_t - \mu)(r_{t-1} - \mu)}{\sum_{t=1}^N (r_t - \mu)^2}$$

Em SP, a transição de \( AC(1) \) de valores negativos (controle) para positivos (lentidão) é o sinal preditivo mestre do colapso de leitos.

class UltraHealthEngine:
    def __init__(self, series):
        self.series = np.array(series, dtype=float)
        # Log-returns: Captura a taxa de aceleração do sistema (momentum)
        self.returns = np.diff(np.log(self.series + 1e-6))

    def dfa_alpha(self, ts):
        """Detrended Fluctuation Analysis (DFA): Mede a memória fractal."""
        if len(ts) < 20: return 0.5
        y = np.cumsum(ts - np.mean(ts))
        n_vals = np.unique(np.logspace(1, 2, 8).astype(int))
        rms_vals = []
        for n in n_vals:
            segments = len(y) // n
            if segments == 0: continue
            # Detrending linear em cada segmento
            rms = np.sqrt(np.mean([np.mean((y[i*n:(i+1)*n] - np.polyval(np.polyfit(np.arange(n), y[i*n:(i+1)*n], 1), np.arange(n)))**2) for i in range(segments)]))
            rms_vals.append(rms)
        if len(rms_vals) < 2: return 0.5
        return np.polyfit(np.log(n_vals[:len(rms_vals)]), np.log(rms_vals), 1)[0]

    def lempel_ziv(self, ts):
        """Mede a inovação algorítmica (Complexidade LZ)."""
        if len(ts) < 5: return 0
        s = "".join(['1' if x > np.median(ts) else '0' for x in ts])
        n = len(s)
        i, u, v, w, c = 1, 1, 1, 1, 1
        while v + w <= n:
            if s[i:i+v] == s[i+w:i+v+w]: v += 1
            else: w += 1; c += 1; v = 1
        return (c * math.log2(n)) / n

    def fisher_info(self, ts):
        """Informação de Fisher: Mede o nível de ordem e nitidez estatística."""
        if len(ts) < 10: return 0
        h, _ = np.histogram(ts, bins=20, density=True)
        prob = h / (np.sum(h) + 1e-12)
        return 4 * np.sum(np.diff(np.sqrt(prob))**2)

    def permutation_entropy(self, ts, d=3):
        """Entropia de Permutação: Previsibilidade baseada na ordem dos dias."""
        ts_arr = np.array(ts)
        if len(ts_arr) < d: return 1.0
        patterns = [tuple(np.argsort(ts_arr[i:i+d])) for i in range(len(ts_arr)-d+1)]
        _, counts = np.unique(patterns, axis=0, return_counts=True)
        p = counts / counts.sum()
        return -np.sum(p * np.log2(p)) / np.log2(math.factorial(d))

    def reconstruct_potential(self):
        """U(x): Paisagem de estabilidade baseada na distribuição de Boltzmann."""
        hist, bins = np.histogram(self.returns, bins=30, density=True)
        centers = (bins[:-1] + bins[1:]) / 2
        sigma = np.std(self.returns)
        p = hist + 1e-12
        U = - (sigma**2 / 2) * np.log(p)
        return centers, U

    def multiscale_entropy(self, max_scale=10):
        """Analisa a entropia em diferentes escalas temporais."""
        ts = self.returns
        n = len(ts)
        entropies = []
        for scale in range(1, max_scale + 1):
            if n // scale < 5:
                entropies.append(1.0)
                continue
            rescaled = ts[:n - n % scale].reshape(-1, scale).mean(axis=1)
            entropies.append(self.permutation_entropy(rescaled))
        return np.array(entropies)

    def get_rolling_metrics(self, window=14):
        """Calcula a evolução temporal de todos os sensores de complexidade."""
        metrics = {
            'resilience': [], 'alpha': [], 'lz': [], 'fisher': [],
            'ac1': [], 'vol': [], 'skew': [], 'kurt': []
        }
        for i in range(window, len(self.returns)):
            slice_ts = self.returns[i-window:i]
            a = self.dfa_alpha(slice_ts)
            lz = self.lempel_ziv(slice_ts)
            f = self.fisher_info(slice_ts)
            ac1 = pd.Series(slice_ts).autocorr(lag=1)
            vol = np.std(slice_ts)

            # Resilience Index: Combinação de memória, lentidão e volatilidade
            ac1_norm = ((ac1 if not np.isnan(ac1) else 0) + 1) / 2
            risk = (ac1_norm + a + (np.log10(vol**2+1e-9)+4)/4) / 3

            metrics['resilience'].append(1 - risk)
            metrics['alpha'].append(a)
            metrics['lz'].append(lz)
            metrics['fisher'].append(f)
            metrics['ac1'].append(ac1)
            metrics['vol'].append(vol)
            metrics['skew'].append(stats.skew(slice_ts))
            metrics['kurt'].append(stats.kurtosis(slice_ts))

        return {k: np.array(v) for k, v in metrics.items()}

@st.cache_data
def load_data(uf, ano, mes, cid, agg):
    try:
        sih = SIH(); sih.load()
        files = sih.get_files(group='RD', uf=uf, year=ano, month=mes)
        if not files: return pd.Series()
        df = sih.download(files[0]).to_dataframe()
        if cid:
            df = df[df['DIAG_PRINC'].str.startswith(cid.upper(), na=False)]
        df['DT_INTER'] = pd.to_datetime(df['DT_INTER'], errors='coerce')
        ts = df.groupby('DT_INTER').size().rename('internacoes').sort_index()
        if agg == 'W': ts = ts.resample('W').sum()
        return ts
    except Exception as e:
        st.error(f"Erro ao descarregar dados: {e}")
        return pd.Series()
